# Webcam 모델 학습 실험 분석 보고서
Webcam(CCTV) 기반 불법 주정차 1차 감지 시스템 | WC-v1~v4 실험 종합 정리

---

## ① 성능 지표 우선순위 — 왜 이 지표를 보는가

Webcam의 역할은 **5분 초과 차량을 감지하여 AMR 출동 좌표를 발행**하는 것이다.  
핵심 로직은 **차량 탐지 → 최초 탐지 timestamp 기록 → 300초 카운트 → 불법 주정차 판정 → AMR 출동**이다.  
탐지를 한 번이라도 놓치면 타이머가 리셋되기 때문에 지표 선택이 중요하다.

| 순위 | 지표 | 왜 이 지표가 중요한가 | 이 시스템에서 실패하면? |
|---|---|---|---|
| ★★★ | Recall ↑ | 탐지를 한 번이라도 놓치면 5분 타이머가 리셋됨. 연속적으로 탐지를 유지해야 단속 판정이 가능 | 차량 미탐지 → 타이머 리셋 → 단속 조건 미충족 → AMR 출동 없음 |
| ★★☆ | mAP@50-95 ↑ | bbox가 얼마나 정밀하게 그려지는지. CCTV 화면 내 pixel 좌표를 실세계 좌표로 변환하는 정확도에 직결 | bbox가 느슨함 → 좌표 변환 오차 증가 → AMR이 엉뚱한 위치로 출동 |
| ★★☆ | Precision ↑ | 오탐(없는데 있다고 탐지) 비율. AMR2가 현장 도착 후 차량 존재 여부를 재확인하는 구조로 완충되나, 헛출동 비용 발생 | 오단속 3% 미만 요구사항 위반 위험. 단, 현장 재확인으로 대부분 걸러짐 |
| ★☆☆ | mAP@50 ↑ | 종합 탐지 성능 참고 지표 | — |
| 필수 | 추론 FPS ≥ 1 | Webcam은 1FPS 설정. 각 프레임에서 정확한 탐지가 필요 | 탐지 누락 → 타이머 리셋 반복 → 단속 불가 |

> ※ val/cls_loss는 Webcam이 car 단일 클래스이므로 AMR 대비 중요도 낮음. val/box_loss와 함께 수렴 안정성 참고 지표로 활용

## ② 모델별 설정 비교 — 무엇을 바꿨나

WC-v2(증강 없음, YOLOv8n)를 기준으로 각 모델에서 달라진 설정을 표시했다. **굵게** = 기준 대비 변경된 항목.

| 설정 항목 | WC-v1 (YOLOv8n+증강) | WC-v2 (YOLOv8n) | WC-v3 (YOLOv11n) | WC-v4 (YOLO26n) |
|---|---|---|---|---|
| **베이스 모델** | YOLOv8n | YOLOv8n | **YOLOv11n** | **YOLO26n** |
| **batch size** | 16 | 16 | 16 | 16 |
| **epochs** | 100 | 100 | 100 | 100 |
| **imgsz** | 640 | 640 | 640 | 640 |
| **optimizer** | Adam | Adam | Adam | Adam |
| **lr0** | 0.01 | 0.01 | 0.01 ⚠️ | 0.01 |
| **lrf** | 0.01 | 0.01 | 0.01 | 0.01 |
| **patience** | 20 | 20 | 20 | 20 |
| **이미지 증강** | **있음** | 없음 | 없음 | 없음 |
| **탐지 클래스** | car | car | car | car |

> ⚠️ WC-v3: YOLOv11n은 개선된 아키텍처로 낮은 lr이 필요한데 lr0=0.01 그대로 적용 → gradient exploding 발생

## ④ 실험 의도 & 결과 해석 — 왜 이 설정을 썼고 어떤 결론이 났나

### WC-v1 — YOLOv8n + 이미지 증강

- **실험 의도:** 도로 고정 카메라 특성상 다양한 조도·각도 변화에 대응하기 위해 이미지 증강 적용
- **기준 대비 변경:** 이미지 증강 추가 (나머지 동일)
- **결과:** Recall 0.9811 (WC-v2 1.0000보다 낮음) / background FP=5개로 오탐은 가장 적음 / 학습 시간 169s로 가장 길음
- **결론:** 이 데이터셋에서는 증강이 오히려 Recall을 낮추는 역효과 발생. 오탐 억제 효과는 있으나 Recall이 최우선인 이 시스템에서는 채택 이유 없음. → **증강 방향 폐기**

---

### WC-v2 — YOLOv8n, 증강 없음

- **실험 의도:** 증강 없이 순수 YOLOv8n 기본 성능 확인. 기준선 역할
- **기준 대비 변경:** 없음 (기준)
- **결과:** Recall **1.0000** / mAP@50 0.9950으로 수치상 최고 / 단, background FP=**12개**로 4개 중 최다 오탐
- **결론:** Recall 1.0은 검증셋이 작은 환경에서 과적합 신호일 가능성 있음. 오탐이 가장 많고 mAP@50-95(0.6452)가 WC-v4(0.6858)보다 낮아 bbox 정밀도에서 뒤처짐. → **불안정 요소 존재, 채택 보류**

---

### WC-v3 — YOLOv11n ❌ 학습 실패

- **실험 의도:** 더 발전된 아키텍처(YOLOv11n)로 전반적 성능 향상 기대
- **기준 대비 변경:** 베이스 모델 `YOLOv8n → YOLOv11n` (lr0=0.01 그대로 유지)
- **결과:**
  ```
  epoch 4: val_cls_loss = inf
  epoch 5: val_cls_loss = inf
  epoch 6: val_cls_loss = nan  ← 완전 발산
  ```
  Confusion Matrix에서 차량 112개를 **전부 background로 예측**. F1 curve 최대값 0.04로 사실상 랜덤 수준
- **결론:** YOLOv11n은 개선된 아키텍처 특성상 낮은 learning rate가 필요한데 lr0=0.01을 그대로 적용해 gradient exploding 발생. 아키텍처 자체의 문제가 아님. → **lr0=0.001로 재학습 시도 권장**

---

### WC-v4 — YOLO26n 🏆 최종 채택

- **실험 의도:** 더 가벼우면서 정밀도가 높은 아키텍처(YOLO26n)로 bbox 품질 향상 기대
- **기준 대비 변경:** 베이스 모델 `YOLOv8n → YOLO26n` (나머지 동일)
- **결과:** Recall 0.9901 / **mAP@50-95 0.6858 최고** ✅ / val/box_loss 1.1501 최저 ✅ / val/cls_loss 0.3522 최저 ✅ / background FP=10개 (WC-v2 12개보다 적음)
- **결론:** 수치상 최고 지표인 WC-v2의 Recall 1.0은 과적합 의심인 반면, WC-v4는 Recall 0.9901로 안정적이면서 가장 중요한 bbox 정밀도(mAP@50-95)와 loss 수렴 품질이 모두 최우수. 균형 잡힌 최고 모델. → **최종 채택**

---

## ⑤ 최종 결론 및 다음 스텝

### 채택 모델

- **WC-v4 (YOLO26n) 사용** — mAP@50-95 0.6858 최고, val loss 전체 최저, Recall·Precision 균형 우수
- **conf threshold → 0.60~0.65 즉시 적용** (F1 최적점 0.608 기준. 현재 0.87은 Recall을 스스로 낮추는 설정)

### 실험에서 배운 것

- 이미지 증강(WC-v1): 이 데이터셋에서는 오히려 Recall 저하 → 증강 방향 폐기
- WC-v2 Recall=1.0: 수치상 최고지만 과적합 의심 + 오탐 최다 → 신뢰도 낮음
- YOLOv11n(WC-v3): lr0=0.01로 gradient exploding 발생 — 아키텍처 문제 아님, lr 문제
- mAP@50-95 차이(WC-v4=0.686 vs WC-v2=0.645)가 CCTV 좌표 추출 정확도에 실질적 영향

### ⚠️ 미완료 항목 및 권장 후속 작업

- **CCTV conf threshold 0.87 → 0.60~0.65 즉시 변경 필요**
  - F1 최적점(0.608)보다 훨씬 높아 Recall을 스스로 제한하는 설정
  - 시스템 요구사항 탐지율 95% 이상 미충족 위험

- **WC-v3 (YOLOv11n) 재학습 권장**
  ```python
  # 재학습 시 lr0 변경
  model.train(data=DATA_PATH, epochs=100, imgsz=640,
              lr0=0.001,  # 0.01 → 0.001로 10배 낮춤
              lrf=0.01, ...)
  ```
  아키텍처 자체는 우수하므로 lr 수정 후 재시도 가치 있음

- **실도로 환경 데이터 추가 수집** — 야간, 우천, 대형차(버스·트럭) 가림 케이스
